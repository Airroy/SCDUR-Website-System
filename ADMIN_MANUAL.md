## 1. ภาพรวม Admin Panel

Admin Panel คือส่วนจัดการข้อมูลของระบบ SCD สำหรับผู้มีสิทธิ์เท่านั้น ผู้ดูแลระบบสามารถ:

- สร้าง แก้ไข ลบ **ปี SCD**
- จัดการ **Banner** รูปภาพสไลด์พร้อมลิงก์
- จัดการ **ตัวชี้วัด (Content)** ในรูปแบบโครงสร้างโฟลเดอร์
- จัดการ **รายงาน SCD** (ไฟล์ PDF ประจำปี)
- จัดการ **ประกาศ** และ **คำสั่ง** ในรูปแบบโครงสร้างโฟลเดอร์
- จัดการ **โปรไฟล์** และรหัสผ่านของตนเอง

### URL ของ Admin Panel

| URL | หน้า |
|-----|------|
| `http://localhost/login` | หน้าเข้าสู่ระบบ |
| `http://localhost/admin/dashboard` | Dashboard |
| `http://localhost/admin/years` | จัดการปี SCD |
| `http://localhost/admin/banners/{ปี?}` | จัดการ Banner |
| `http://localhost/admin/contents/{ปี?}` | จัดการตัวชี้วัด |
| `http://localhost/admin/reports/{ปี?}` | จัดการรายงาน |
| `http://localhost/admin/announcements/{ปี?}` | จัดการประกาศ |
| `http://localhost/admin/directives/{ปี?}` | จัดการคำสั่ง |
| `http://localhost/admin/profile` | จัดการโปรไฟล์ |

> **หมายเหตุความปลอดภัย:** URL ของ Admin Panel ไม่แสดงบนหน้าเว็บสาธารณะ การเข้าถึงโดยไม่ได้ login จะแสดงหน้า 404 เพื่อปิดบังการมีอยู่ของระบบ

## 2. การเข้าสู่ระบบ

### 2.1 ขั้นตอนการ Login

1. เปิดเบราว์เซอร์ ไปที่ `http://localhost/login`
2. กรอก **อีเมล** และ **รหัสผ่าน**
3. (ไม่บังคับ) ติ๊ก **"จำฉันไว้"** เพื่อ Session คงอยู่นานขึ้น
4. คลิก **"เข้าสู่ระบบ"**
5. ระบบพาไปหน้า Dashboard อัตโนมัติ

### 2.2 การออกจากระบบ

1. คลิกชื่อผู้ใช้มุมบนขวาของหน้า
2. คลิก **"ออกจากระบบ"**
3. ระบบ redirect ไปหน้าเว็บหลัก

### 2.3 ข้อควรระวัง

- บัญชีถูกสร้างโดย IT เท่านั้น — **ไม่มีการสมัครสมาชิกเอง**
- Login ผิดพลาดเกิน **5 ครั้ง** ระบบจะล็อคชั่วคราว (รอ 1 นาที)
- ถ้าลืมรหัสผ่าน ต้องติดต่อ IT เพื่อ Reset (ไม่มีฟีเจอร์ส่งอีเมล Reset)
- **บัญชี Admin ทุกบัญชีมีสิทธิ์เท่ากัน** — ไม่มีระบบแยก Role/Permission

## 3. Dashboard

**URL:** `http://localhost/admin/dashboard`

แสดงภาพรวมสถิติของระบบในขณะนั้น:

| การ์ดสถิติ | ข้อมูลที่แสดง |
|----------|-------------|
| **ปี SCD ทั้งหมด** | จำนวนปีทั้งหมดในระบบ (ทั้งที่เผยแพร่และซ่อน) |
| **ปีที่เผยแพร่** | จำนวนปีที่ผู้เยี่ยมชมมองเห็นได้ |
| **รายงานทั้งหมด** | จำนวนไฟล์รายงาน PDF ที่อัปโหลดแล้ว |
| **Banner ทั้งหมด** | จำนวน Banner รูปภาพในระบบ |

### เมนูนำทาง (Sidebar)

Sidebar ด้านซ้ายมือมีลิงก์ไปยังทุกส่วนของ Admin Panel พร้อมแสดงชื่อหน้าปัจจุบัน

## 4. จัดการปี SCD

**URL:** `http://localhost/admin/years`

ปี SCD คือหน่วยหลักของระบบ ข้อมูลทุกอย่าง (Banner, Content, รายงาน, ประกาศ, คำสั่ง) ต้องอ้างอิงกับปี ต้องสร้างปีก่อนเสมอ

### 4.1 ดูรายการปี

ตารางแสดง:
- **ปี** (เช่น SCD 2024)
- **วันที่สร้าง**
- **สถานะ** — เผยแพร่ / ซ่อน
- **ปุ่มแก้ไข** และ **ปุ่มลบ**

### 4.2 เพิ่มปีใหม่

1. คลิก **"เพิ่มปีใหม่"** (มุมขวาบน)
2. กรอกข้อมูล:

| ช่อง | คำอธิบาย | หมายเหตุ |
|------|---------|--------|
| **ปี** | ตัวเลข ค.ศ. เช่น `2025` | ไม่ซ้ำกับปีที่มีอยู่ |
| **วันที่สร้าง** | เลือกจาก Date Picker | วันที่อ้างอิงทางการของปีนี้ |
| **สถานะ** | เผยแพร่ / ซ่อน | กำหนดว่าผู้เยี่ยมชมมองเห็นได้หรือไม่ |

3. คลิก **"บันทึก"**

> ปีที่ซ้ำกันจะไม่สามารถบันทึกได้ ระบบแจ้งเตือน Validation

### 4.3 แก้ไขปี

1. คลิกไอคอน **ดินสอ (แก้ไข)** ในแถวของปีที่ต้องการ
2. แก้ไขข้อมูลในฟอร์ม
3. คลิก **"บันทึก"**

### 4.4 เปิด/ปิดการเผยแพร่

คลิกปุ่ม **Toggle** ในคอลัมน์สถานะ ระบบเปลี่ยนทันที:

| สถานะ | สีปุ่ม | ผลต่อหน้าเว็บ |
|------|------|-------------|
| **เผยแพร่** | เขียว | ผู้เยี่ยมชมมองเห็นข้อมูลของปีนี้ |
| **ซ่อน** | เทา | ซ่อนข้อมูลทั้งหมดของปีนี้จากสาธารณะ |

> การซ่อนปีซ่อน **ทุกอย่าง** ในปีนั้นพร้อมกัน: Banner, Content, รายงาน, ประกาศ, คำสั่ง

### 4.5 ลบปี

1. คลิกไอคอน **ถังขยะ (ลบ)** ในแถวของปีที่ต้องการ
2. ยืนยันในป๊อปอัพยืนยัน

> **คำเตือน:** การลบปีลบ **ข้อมูลทั้งหมด** ของปีนั้นถาวร — Banner, Content, รายงาน, ประกาศ, คำสั่ง รวมถึงไฟล์ใน Storage ไม่สามารถกู้คืนได้

## 5. จัดการ Banner

**URL:** `http://localhost/admin/banners/{ปี?}`

Banner คือรูปภาพที่แสดงใน Slider บนหน้าเว็บ แต่ละ Banner สังกัดปี SCD หนึ่งปี

### 5.1 เลือกปีที่ต้องการจัดการ

เลือกปีจากแถบเลือกปีที่ด้านบนของหน้า URL จะเปลี่ยนเป็น `/admin/banners/{ปี}`

### 5.2 เพิ่ม Banner

1. คลิก **"เพิ่ม Banner"**
2. กรอกข้อมูล:

| ช่อง | คำอธิบาย | บังคับ |
|------|---------|--------|
| **รูปภาพ** | อัปโหลด JPG/PNG สูงสุด 100 MB | ✅ |
| **หมวดหมู่** | `0` = Slider หลัก, `1` = Slider รอง | ✅ |
| **ประเภทลิงก์** | ไม่มีลิงก์ / URL / PDF | ✅ |
| **URL** | ระบุ URL ปลายทาง (กรณีเลือก URL) | เฉพาะ URL |
| **ไฟล์ PDF** | อัปโหลด PDF สูงสุด 100 MB (กรณีเลือก PDF) | เฉพาะ PDF |

3. ใช้ **Image Cropper** เพื่อปรับขนาดรูปภาพให้พอดีกับ Slider
4. คลิก **"บันทึก"**

**ประเภทลิงก์:**
- **ไม่มีลิงก์** — ภาพแสดงเฉยๆ ไม่มีการกดคลิก
- **URL** — เมื่อผู้เยี่ยมชมคลิกภาพจะเปิด URL ที่กำหนด
- **PDF** — เมื่อผู้เยี่ยมชมคลิกภาพจะเปิดไฟล์ PDF ที่อัปโหลด

### 5.3 แก้ไข Banner

1. คลิก **"แก้ไข"** ข้างรายการ
2. แก้ไขข้อมูลที่ต้องการ
   - ถ้าไม่เลือกรูปใหม่ รูปเดิมจะยังคงอยู่
   - ถ้าไม่เลือก PDF ใหม่ PDF เดิมจะยังคงอยู่
3. คลิก **"บันทึก"**

### 5.4 จัดลำดับ Banner

1. คลิก **"จัดลำดับ"** (เลือกหมวดหมู่ที่ต้องการก่อน)
2. ลากรายการ (Drag & Drop) เพื่อเรียงลำดับ
3. คลิก **"บันทึกลำดับ"**

> ลำดับจะส่งผลต่อลำดับภาพใน Slider บนหน้าเว็บทันที

### 5.5 ลบ Banner

1. คลิก **"ลบ"** ข้างรายการ
2. ยืนยันในป๊อปอัพ

> ระบบลบทั้งข้อมูลในฐานข้อมูลและไฟล์รูปภาพ/PDF ใน Storage พร้อมกัน

## 6. จัดการตัวชี้วัด (Content)

**URL:** `http://localhost/admin/contents/{ปี?}`

Content คือตัวชี้วัดและเนื้อหาหลักของ SCD ที่แสดงในหน้าแรก จัดเก็บเป็น **โครงสร้างโฟลเดอร์ (Tree)**

### 6.1 โครงสร้าง Tree

```
📁 ตัวชี้วัด A (root)
├── 📁 ตัวชี้วัดย่อย A.1
│   ├── 📄 เอกสาร A.1.1.pdf
│   └── 📄 เอกสาร A.1.2.pdf
└── 📄 เอกสารรวม A.pdf

📁 ตัวชี้วัด B (root)
└── 📄 เอกสาร B.pdf
```

### 6.2 การนำทางภายใน Content

- คลิก **📁 โฟลเดอร์** เพื่อเข้าดูรายการภายใน
- ดู **Breadcrumbs** ด้านบนเพื่อรู้ตำแหน่งปัจจุบัน เช่น `ตัวชี้วัด A > ตัวชี้วัดย่อย A.1`
- คลิก Breadcrumb เพื่อย้อนกลับ

### 6.3 เพิ่ม Content

1. เลือกปีจากแถบเลือกปี
2. (ถ้าต้องการเพิ่มภายในโฟลเดอร์) เปิดโฟลเดอร์นั้นก่อน
3. คลิก **"เพิ่มโฟลเดอร์"** หรือ **"เพิ่มไฟล์"**

**โฟลเดอร์ (สร้างหมวดหมู่):**

| ช่อง | คำอธิบาย | บังคับ |
|------|---------|--------|
| **ชื่อ** | ชื่อโฟลเดอร์ที่แสดง | ✅ |
| **รูปภาพ** | รูปประจำโฟลเดอร์ (JPG/PNG) สูงสุด 100 MB | — |
| **แสดง/ซ่อน** | Toggle เปิด-ปิดการแสดงผลบนหน้าเว็บ | — |

**ไฟล์ (อัปโหลดเอกสาร):**

| ช่อง | คำอธิบาย | บังคับ |
|------|---------|--------|
| **ชื่อ** | ชื่อรายการที่แสดง | ✅ |
| **ไฟล์ PDF** | ไฟล์เอกสาร PDF สูงสุด 100 MB | ✅ |
| **รูปภาพ** | รูปประกอบ (JPG/PNG) สูงสุด 100 MB | — |
| **แสดง/ซ่อน** | Toggle เปิด-ปิดการแสดงผลบนหน้าเว็บ | — |

4. คลิก **"บันทึก"**

### 6.4 แก้ไข Content

1. คลิก **"แก้ไข"** ข้างรายการ
2. แก้ไขข้อมูล
3. คลิก **"บันทึก"**

> ถ้าไม่เลือกไฟล์ใหม่ ไฟล์เดิมจะยังคงอยู่

### 6.5 แสดง/ซ่อน Content

- คลิกปุ่ม **Toggle** ในคอลัมน์สถานะ
- รายการที่ซ่อนยังคงอยู่ในฐานข้อมูล Admin ยังเห็น แต่ผู้เยี่ยมชมจะไม่เห็นบนหน้าเว็บ

### 6.6 จัดลำดับ Content

1. คลิก **"จัดลำดับ"**
2. ลากรายการ (Drag & Drop) เพื่อเรียงลำดับใหม่
3. คลิก **"บันทึกลำดับ"**

> ลำดับใช้งานกับรายการในระดับเดียวกัน (ระดับ root หรือภายใน folder เดียวกัน)

### 6.7 ลบ Content

1. คลิก **"ลบ"** ข้างรายการ
2. ยืนยันในป๊อปอัพ

> **คำเตือน:** การลบโฟลเดอร์ลบ **ทุกรายการและไฟล์ภายใน** โฟลเดอร์นั้นถาวร

## 7. จัดการรายงาน SCD

**URL:** `http://localhost/admin/reports/{ปี?}`

รายงาน SCD คือไฟล์ PDF รายงานผลการจัดอันดับประจำปี **แต่ละปีมีได้เพียง 1 ไฟล์**

### 7.1 ดูรายการรายงาน

ตารางแสดง:
- **ชื่อไฟล์** รายงาน
- **จำนวนการดู** (View Count) — นับโดยอัตโนมัติเมื่อผู้เยี่ยมชมเปิดดู
- **วันที่อัปโหลด**

### 7.2 อัปโหลดรายงาน

1. เลือกปีจากแถบเลือกปี
2. คลิก **"อัปโหลดรายงาน"**
3. เลือกไฟล์ PDF (สูงสุด 100 MB)
4. คลิก **"อัปโหลด"**

> ถ้าปีนั้นมีรายงานแล้ว ต้องลบรายงานเก่าก่อนจึงจะอัปโหลดใหม่ได้ (1 ปี = 1 รายงาน)

### 7.3 แก้ไขรายงาน (อัปเดตไฟล์)

1. คลิก **"แก้ไข"** ข้างรายการ
2. เลือกไฟล์ PDF ใหม่ (ถ้าไม่เลือก ไฟล์เดิมยังคงอยู่)
3. คลิก **"บันทึก"**

> ระบบจะลบไฟล์เก่าออกจาก Storage โดยอัตโนมัติเมื่ออัปโหลดไฟล์ใหม่

### 7.4 ลบรายงาน

1. คลิก **"ลบ"** ข้างรายการ
2. ยืนยันในป๊อปอัพ
3. ระบบลบทั้งข้อมูลและไฟล์ PDF ออกจาก Storage

## 8. จัดการประกาศ

**URL:** `http://localhost/admin/announcements/{ปี?}`  
(โฟลเดอร์: `http://localhost/admin/announcements/{ปี}/folder/{folderId}`)

ประกาศ (Announcements) จัดเก็บในรูปแบบ **โครงสร้างโฟลเดอร์ (Tree)** สังกัดปี SCD

### 8.1 โครงสร้าง Tree

```
📁 ไตรมาสที่ 1
├── 📄 ประกาศฉบับที่ 1.pdf
└── 📄 ประกาศฉบับที่ 2.pdf
📄 ประกาศกลางปี.pdf
```

### 8.2 การนำทาง

- คลิก **📁 โฟลเดอร์** เพื่อเข้าดูรายการภายใน (URL เปลี่ยนเป็น `.../folder/{id}`)
- Breadcrumbs แสดงตำแหน่งปัจจุบัน คลิกเพื่อย้อนกลับ

### 8.3 เพิ่มประกาศ

1. เลือกปีจากแถบเลือกปี
2. (ถ้าต้องการเพิ่มภายในโฟลเดอร์) เปิดโฟลเดอร์นั้นก่อน
3. คลิก **"เพิ่มโฟลเดอร์"** หรือ **"เพิ่มไฟล์"**
4. กรอกข้อมูล (เหมือนกับ Content — ดูหัวข้อ 6.3)
5. คลิก **"บันทึก"**

### 8.4 แก้ไขประกาศ

1. คลิก **"แก้ไข"** ข้างรายการ
2. แก้ไขข้อมูล
3. คลิก **"บันทึก"**

### 8.5 แสดง/ซ่อนประกาศ

- คลิกปุ่ม **Toggle** ในคอลัมน์สถานะ
- รายการที่ซ่อนจะไม่แสดงบนหน้าเว็บสาธารณะ แต่ Admin ยังเห็นในตาราง

### 8.6 จัดลำดับประกาศ

1. คลิก **"จัดลำดับ"**
2. ลากรายการ (Drag & Drop) เพื่อเรียงใหม่
3. คลิก **"บันทึกลำดับ"**

### 8.7 ลบประกาศ

1. คลิก **"ลบ"** ข้างรายการ
2. ยืนยันในป๊อปอัพ

> **คำเตือน:** การลบโฟลเดอร์ลบไฟล์ทั้งหมดภายในโฟลเดอร์ถาวร

## 9. จัดการคำสั่ง

**URL:** `http://localhost/admin/directives/{ปี?}`  
(โฟลเดอร์: `http://localhost/admin/directives/{ปี}/folder/{folderId}`)

คำสั่ง (Directives) มีการจัดการ **เหมือนกับประกาศทุกขั้นตอน** ต่างกันเพียง:
- เก็บข้อมูลแยกตาราง (`scd_directives`) เพื่อแยกประเภทเอกสาร
- URL ใช้ `/admin/directives/` แทน `/admin/announcements/`

> ดูขั้นตอนการจัดการทั้งหมดที่ [หัวข้อ 8 จัดการประกาศ](#8-จัดการประกาศ)

## 10. จัดการโปรไฟล์

**URL:** `http://localhost/admin/profile`

### 10.1 แก้ไขข้อมูลส่วนตัว

1. แก้ไข **ชื่อ** และ/หรือ **อีเมล**
2. คลิก **"บันทึก"**

> หากเปลี่ยนอีเมล ระบบส่งลิงก์ยืนยันไปยังอีเมลใหม่ ต้องยืนยันก่อนใช้งาน

### 10.2 เปลี่ยนรหัสผ่าน

1. กรอก **รหัสผ่านปัจจุบัน**
2. กรอก **รหัสผ่านใหม่** (ขั้นต่ำ 8 ตัวอักษร)
3. กรอก **ยืนยันรหัสผ่านใหม่**
4. คลิก **"เปลี่ยนรหัสผ่าน"**

**ข้อกำหนด:**
- รหัสผ่านขั้นต่ำ 8 ตัวอักษร
- รหัสผ่านใหม่ทั้งสองช่องต้องตรงกัน
- รหัสผ่านปัจจุบันต้องถูกต้อง

### 10.3 ลบบัญชีผู้ใช้

> **คำเตือน:** การลบบัญชีเป็นการถาวร ไม่สามารถกู้คืนได้ ต้องสร้างบัญชีใหม่ผ่าน IT เท่านั้น

1. คลิก **"ลบบัญชี"** ด้านล่างหน้า
2. กรอกรหัสผ่านเพื่อยืนยัน
3. คลิก **"ยืนยันการลบ"**
4. ระบบออกจากระบบและลบบัญชีอัตโนมัติ

## 11. ข้อจำกัดและข้อควรระวัง

### 11.1 ข้อจำกัดไฟล์อัปโหลด

| ประเภทไฟล์ | ขนาดสูงสุด | รูปแบบที่รองรับ |
|------------|------------|----------------|
| รูปภาพ (Banner, Content, ประกาศ) | 100 MB | JPG, PNG, WEBP, GIF |
| ไฟล์ PDF | 100 MB | PDF เท่านั้น |

### 11.2 ผลของการลบข้อมูล

| สิ่งที่ลบ | ผลที่เกิดขึ้น |
|----------|-------------|
| **ปี SCD** | ลบ Banner, Content, รายงาน, ประกาศ, คำสั่ง ทั้งหมดของปีนั้น รวมถึงไฟล์ใน Storage |
| **โฟลเดอร์** | ลบรายการและไฟล์ทั้งหมดภายในโฟลเดอร์ |
| **ไฟล์/Banner** | ลบข้อมูลและไฟล์จาก Storage ถาวร |
| **บัญชีผู้ใช้** | ลบบัญชีถาวร ต้องให้ IT สร้างใหม่ |

> ระบบ **ไม่มี Recycle Bin** — การลบทุกอย่างเป็นถาวร ควร Backup ข้อมูลสำคัญก่อนลบเสมอ

### 11.3 ข้อจำกัดด้านบัญชีผู้ใช้

- ไม่มีการสมัครสมาชิกเอง — ต้องให้ IT สร้างผ่าน Command Line
- ไม่มีระบบ Reset Password ผ่านอีเมล — ติดต่อ IT เท่านั้น
- บัญชีทุกบัญชีมีสิทธิ์เท่ากัน (Full Admin) — ไม่มี Role แยก

## 12. การแก้ไขปัญหาเบื้องต้น

### ปัญหา: เข้า Admin ไม่ได้ (แสดง 404)

ระบบแสดง 404 เมื่อไม่ได้ Login เพื่อปลอดภัย

**วิธีแก้:**
1. ไปที่ `http://localhost/login`
2. Login ใหม่อีกครั้ง

---

### ปัญหา: Login ไม่ได้ (Too Many Attempts)

**สาเหตุ:** Login ผิดพลาดเกิน 5 ครั้ง  
**วิธีแก้:** รอ 1 นาทีแล้วลองใหม่ หรือแจ้ง IT ล้าง Rate Limit

---

### ปัญหา: อัปโหลดไฟล์ไม่สำเร็จ

ตรวจสอบ:
1. ขนาดไฟล์เกิน 100 MB หรือไม่?
2. นามสกุลไฟล์ถูกต้อง? (ต้องเป็น PDF, JPG, PNG เท่านั้น)
3. การเชื่อมต่ออินเทอร์เน็ตปกติ?

หากยังไม่ได้ผล แจ้ง IT ตรวจสอบ `storage/logs/laravel.log`

---

### ปัญหา: รูปภาพไม่แสดงผล

**สาเหตุ:** Storage symlink ขาด  
**วิธีแก้:** แจ้ง IT รันคำสั่ง:
```bash
./vendor/bin/sail artisan storage:link
```

---

### ปัญหา: ข้อมูลไม่อัปเดตหลังบันทึก

**วิธีแก้:**
1. กด `Ctrl + F5` (Windows) หรือ `Cmd + Shift + R` (Mac) เพื่อ Hard Refresh
2. ถ้ายังไม่หาย แจ้ง IT ล้าง Cache:
```bash
./vendor/bin/sail artisan cache:clear
./vendor/bin/sail artisan view:clear
```

---

### ปัญหา: เว็บขึ้น Error 500

แจ้ง IT ตรวจสอบ logs:
```bash
./vendor/bin/sail logs
# หรือดูที่ storage/logs/laravel.log
```

## 13. คำถามที่พบบ่อย

**Q: สามารถเพิ่มบัญชี Admin เองได้ไหม?**  
A: ไม่ได้ ต้องให้ IT รันคำสั่ง `./vendor/bin/sail artisan admin:create`

---

**Q: ลืมรหัสผ่านต้องทำอย่างไร?**  
A: ติดต่อ IT เพื่อ Reset รหัสผ่านหรือสร้างบัญชีใหม่ ระบบไม่มีฟีเจอร์ส่งอีเมล Reset

---

**Q: ปีที่ "ซ่อน" ยังแก้ไขได้ใน Admin ไหม?**  
A: ได้ การซ่อนปีเป็นแค่การซ่อนจากผู้เยี่ยมชม Admin ยังจัดการข้อมูลได้ตามปกติ

---

**Q: ไฟล์ที่ลบแล้วกู้คืนได้ไหม?**  
A: ไม่ได้ การลบเป็นถาวร ระบบไม่มี Recycle Bin

---

**Q: Banner Slider เพิ่มได้มากแค่ไหน?**  
A: ไม่จำกัดจำนวน แต่แนะนำ 5–7 ภาพเพื่อ UX ที่ดี

---

**Q: ทำไมต้องใช้ Image Cropper ตอนอัปโหลดรูป Banner?**  
A: เพื่อให้รูปมีสัดส่วนที่เหมาะสมกับ Slider โดยอัตโนมัติ ป้องกันภาพผิดสัดส่วน

---

**Q: View Count และ Download Count นับอย่างไร?**  
A: นับ 1 ครั้งต่อ Session ต่อไฟล์ ถ้าผู้เยี่ยมชมเปิดไฟล์เดิมซ้ำในการเยี่ยมชมครั้งเดียวกัน จะไม่นับซ้ำ

---

**Q: สามารถเข้า Admin จากนอกองค์กรได้ไหม?**  
A: ขึ้นกับการตั้งค่า Network ปกติต้องอยู่ใน Network ขององค์กรหรือใช้ VPN

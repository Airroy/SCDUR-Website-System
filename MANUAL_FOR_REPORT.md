# ภาคผนวก — คู่มือการใช้งานระบบ SCD

**ระบบจัดอันดับมหาวิทยาลัยด้านการพัฒนาชุมชนอย่างยั่งยืน**  
Sustainable Community Development University Ranking System  
มหาวิทยาลัยราชภัฏพระนครศรีอยุธยา

---

## ภาคผนวก ก — คู่มือการใช้งานสำหรับผู้เยี่ยมชม

### ก.1 ภาพรวมระบบ

ระบบ SCD เป็นเว็บไซต์สำหรับเผยแพร่ข้อมูลการจัดอันดับมหาวิทยาลัยด้านการพัฒนาชุมชนอย่างยั่งยืน (Sustainable Community Development) ของมหาวิทยาลัยราชภัฏพระนครศรีอยุธยา ผู้เยี่ยมชมทั่วไปสามารถใช้งานได้โดยไม่ต้องสมัครสมาชิกหรือเข้าสู่ระบบ

ความสามารถหลักสำหรับผู้เยี่ยมชม ได้แก่ การดูข้อมูลการจัดอันดับ SCD แยกตามปี การดูและดาวน์โหลดรายงาน SCD ประจำปี การดูและดาวน์โหลดประกาศและคำสั่งต่างๆ รวมถึงการดูตัวชี้วัดและเนื้อหาของแต่ละปี

**ที่อยู่เว็บไซต์หลัก**

| หน้าเว็บ | URL |
|---------|-----|
| หน้าแรก | http://localhost/ |
| ข้อมูล SCD ตามปี | http://localhost/scd/{ปี} |
| ประกาศและคำสั่ง | http://localhost/year/{ปี}/announcements-orders |
| ตัวชี้วัดรายหมวด | http://localhost/scd/{ปี}/{หมวด} |
| เกี่ยวกับหน่วยงาน | http://localhost/about |
| ติดต่อเรา | http://localhost/contact |

---

### ก.2 หน้าแรก

หน้าแรกเป็นจุดเริ่มต้นหลักของเว็บไซต์ ประกอบด้วย 4 ส่วนหลัก ดังนี้

**ก.2.1 Banner Slider**

แถบภาพสไลด์ด้านบนสุดของหน้า แสดงภาพประชาสัมพันธ์ของโครงการ SCD โดยมีคุณสมบัติดังนี้
- ภาพสไลด์เปลี่ยนอัตโนมัติ (Auto-slide)
- สามารถกดปุ่มลูกศรซ้าย/ขวา เพื่อเลื่อนภาพด้วยตนเอง
- กดจุดด้านล่างภาพเพื่อข้ามไปยังสไลด์ที่ต้องการ
- บางภาพมีลิงก์แนบ คลิกที่ภาพเพื่อเปิดเนื้อหาหรือไฟล์ PDF ที่เชื่อมโยง

**ก.2.2 แถบเลือกปี SCD**

แถบเลือกปี SCD ที่เผยแพร่แล้ว แสดงอยู่บริเวณด้านบนของเนื้อหาหลัก สามารถคลิกปีที่ต้องการเพื่อดูข้อมูลของปีนั้น URL จะเปลี่ยนเป็น /scd/{ปี} เช่น /scd/2024 ปีที่ยังไม่ได้เผยแพร่จะไม่แสดงในตัวเลือก

**ก.2.3 ประกาศล่าสุด**

แสดงประกาศล่าสุดของปีที่กำลังดูอยู่ สูงสุด 6 รายการ สามารถคลิกที่รายการเพื่อดูรายละเอียดหรือดาวน์โหลดไฟล์แนบ

**ก.2.4 ตัวชี้วัด (Content Sections)**

แสดงหมวดหมู่ตัวชี้วัดและเนื้อหาของ SCD สำหรับปีที่กำลังดูอยู่ แต่ละช่องแสดงชื่อหมวดหมู่พร้อมรูปภาพประกอบ (ถ้ามี) สามารถคลิกที่หมวดหมู่เพื่อเข้าดูตัวชี้วัดและไฟล์เอกสารภายใน

---

### ก.3 หน้า SCD Ranking (เลือกดูตามปี)

URL: http://localhost/scd/{ปี} ตัวอย่างเช่น http://localhost/scd/2024

หน้านี้แสดงข้อมูลการจัดอันดับ SCD ของปีที่ระบุ เนื้อหาเหมือนกับหน้าแรกแต่กรองเฉพาะปีนั้น ประกอบด้วย Banner Slider ของปีนั้น ตัวชี้วัดแยกตามหมวดหมู่ และประกาศที่เกี่ยวข้องกับปีนั้น

การเปลี่ยนปี ทำได้โดยคลิกปุ่มปีบนแถบเลือกปี หรือพิมพ์ URL ตรง เช่น http://localhost/scd/2023 หากพิมพ์ URL ปีที่ไม่มีในระบบหรือยังไม่เผยแพร่ จะแสดงหน้า 404

---

### ก.4 หน้าประกาศและคำสั่ง

URL: http://localhost/year/{ปี}/announcements-orders ตัวอย่างเช่น http://localhost/year/2024/announcements-orders

หน้านี้รวบรวมประกาศ (Announcements) และคำสั่ง (Directives) ของปีที่เลือก แสดงในรูปแบบโครงสร้างโฟลเดอร์ (Tree Structure) ดังนี้

```
โฟลเดอร์หลัก
    โฟลเดอร์ย่อย
        ประกาศฉบับที่ 1.pdf
        ประกาศฉบับที่ 2.pdf
    ประกาศกลางปี.pdf
```

การนำทางในหน้านี้ทำได้โดยคลิกโฟลเดอร์เพื่อเปิดและดูรายการภายใน สำหรับไฟล์ PDF จะมีปุ่มให้เลือก 2 ปุ่ม คือ ปุ่ม "ดูไฟล์" สำหรับเปิดไฟล์ PDF ในหน้าต่างใหม่ของเบราว์เซอร์ และปุ่ม "ดาวน์โหลด" สำหรับบันทึกไฟล์ PDF ลงเครื่อง ระบบนับจำนวนการดูและดาวน์โหลดโดยอัตโนมัติ (นับครั้งเดียวต่อ Session)

---

### ก.5 หน้าตัวชี้วัด (Content Section)

URL: http://localhost/scd/{ปี}/{ชื่อหมวด}

หน้านี้แสดงตัวชี้วัดและเนื้อหาของหมวดหมู่ที่เลือก โครงสร้างการแสดงผลเหมือนกับหน้าประกาศ แต่ละหมวดหมู่อาจมีโฟลเดอร์ย่อยและไฟล์ PDF สามารถคลิกโฟลเดอร์เพื่อขยายดูรายการภายใน และคลิกปุ่ม "ดูไฟล์" หรือ "ดาวน์โหลด" เพื่อเข้าถึงเอกสาร

---

### ก.6 หน้าดาวน์โหลดรายงาน SCD

หน้านี้แสดงรายงานผล SCD ประจำปี (ไฟล์ PDF) แต่ละปีมีรายงานได้ 1 ฉบับ วิธีดูหรือดาวน์โหลดรายงาน มีขั้นตอนดังนี้

1. เลือกปีที่ต้องการจากเมนู
2. คลิก "ดูรายงาน" เพื่อเปิดดูใน PDF Viewer ของเบราว์เซอร์
3. หรือคลิก "ดาวน์โหลด" เพื่อบันทึกไฟล์ลงเครื่อง

ระบบนับจำนวนการดูรายงานโดยอัตโนมัติ

---

### ก.7 หน้าเกี่ยวกับหน่วยงานและหน้าติดต่อเรา

หน้าเกี่ยวกับหน่วยงาน (URL: http://localhost/about) แสดงข้อมูลขององค์กร ประวัติความเป็นมา วิสัยทัศน์ และบทบาทหน้าที่ของหน่วยงาน SCD มหาวิทยาลัยราชภัฏพระนครศรีอยุธยา

หน้าติดต่อเรา (URL: http://localhost/contact) แสดงข้อมูลการติดต่อหน่วยงาน ประกอบด้วยที่อยู่หน่วยงาน หมายเลขโทรศัพท์ อีเมล และแผนที่แสดงที่ตั้ง (Google Maps)

## ภาคผนวก ข — คู่มือการใช้งานสำหรับผู้ดูแลระบบ

### ข.1 ภาพรวม Admin Panel

Admin Panel คือส่วนจัดการข้อมูลของระบบ SCD สำหรับผู้มีสิทธิ์เท่านั้น บัญชีผู้ดูแลระบบต้องได้รับการสร้างโดยผู้ดูแลระบบ IT เท่านั้น ไม่มีฟีเจอร์สมัครสมาชิกด้วยตนเอง

ผู้ดูแลระบบสามารถดำเนินการ ดังนี้
- สร้าง แก้ไข และลบปี SCD
- จัดการ Banner รูปภาพสไลด์พร้อมลิงก์
- จัดการตัวชี้วัด (Content) ในรูปแบบโครงสร้างโฟลเดอร์
- จัดการรายงาน SCD (ไฟล์ PDF ประจำปี)
- จัดการประกาศและคำสั่งในรูปแบบโครงสร้างโฟลเดอร์
- จัดการโปรไฟล์และรหัสผ่านของตนเอง

**ที่อยู่ URL ของ Admin Panel**

| หน้า | URL |
|------|-----|
| หน้าเข้าสู่ระบบ | http://localhost/login |
| Dashboard | http://localhost/admin/dashboard |
| จัดการปี SCD | http://localhost/admin/years |
| จัดการ Banner | http://localhost/admin/banners/{ปี} |
| จัดการตัวชี้วัด | http://localhost/admin/contents/{ปี} |
| จัดการรายงาน | http://localhost/admin/reports/{ปี} |
| จัดการประกาศ | http://localhost/admin/announcements/{ปี} |
| จัดการคำสั่ง | http://localhost/admin/directives/{ปี} |
| จัดการโปรไฟล์ | http://localhost/admin/profile |

หมายเหตุด้านความปลอดภัย: URL ของ Admin Panel ไม่แสดงบนหน้าเว็บสาธารณะ การเข้าถึงโดยไม่ได้เข้าสู่ระบบจะแสดงหน้า 404 เพื่อปิดบังการมีอยู่ของระบบ

---

### ข.2 การเข้าสู่ระบบและออกจากระบบ

**การเข้าสู่ระบบ** มีขั้นตอนดังนี้

1. เปิดเบราว์เซอร์ ไปที่ http://localhost/login
2. กรอกอีเมลและรหัสผ่าน
3. ติ๊ก "จำฉันไว้" (ไม่บังคับ) เพื่อให้ Session คงอยู่นานขึ้น
4. คลิก "เข้าสู่ระบบ"
5. ระบบนำไปหน้า Dashboard โดยอัตโนมัติ

**การออกจากระบบ** มีขั้นตอนดังนี้

1. คลิกชื่อผู้ใช้มุมบนขวาของหน้า
2. คลิก "ออกจากระบบ"
3. ระบบ redirect ไปหน้าเว็บหลัก

**ข้อควรระวัง**
- หากเข้าสู่ระบบผิดพลาดเกิน 5 ครั้ง ระบบจะล็อคชั่วคราว (รอ 1 นาที)
- หากลืมรหัสผ่าน ต้องติดต่อ IT เพื่อ Reset เนื่องจากไม่มีฟีเจอร์ส่งอีเมล Reset
- บัญชี Admin ทุกบัญชีมีสิทธิ์เท่ากัน ไม่มีระบบแยก Role/Permission

---

### ข.3 Dashboard

URL: http://localhost/admin/dashboard

Dashboard แสดงภาพรวมสถิติของระบบ ประกอบด้วย

| สถิติ | ความหมาย |
|------|---------|
| ปี SCD ทั้งหมด | จำนวนปีทั้งหมดในระบบ (ทั้งที่เผยแพร่และซ่อน) |
| ปีที่เผยแพร่ | จำนวนปีที่ผู้เยี่ยมชมมองเห็นได้ |
| รายงานทั้งหมด | จำนวนไฟล์รายงาน PDF ที่อัปโหลดแล้ว |
| Banner ทั้งหมด | จำนวน Banner รูปภาพในระบบ |

เมนูนำทาง (Sidebar) ด้านซ้ายมือมีลิงก์ไปยังทุกส่วนของ Admin Panel

---

### ข.4 จัดการปี SCD

URL: http://localhost/admin/years

ปี SCD คือหน่วยหลักของระบบ ข้อมูลทุกอย่างต้องอ้างอิงกับปี จึงต้องสร้างปีก่อนเสมอ

**ข.4.1 เพิ่มปีใหม่**

1. คลิก "เพิ่มปีใหม่" (มุมขวาบน)
2. กรอกข้อมูล ได้แก่ ปี (ตัวเลข ค.ศ.) วันที่สร้าง และสถานะ (เผยแพร่/ซ่อน)
3. คลิก "บันทึก"

หมายเหตุ: แต่ละปีต้องไม่ซ้ำกัน ปีที่ซ้ำกันจะไม่สามารถบันทึกได้

**ข.4.2 แก้ไขปี**

1. คลิกไอคอนดินสอ (แก้ไข) ในแถวของปีที่ต้องการ
2. แก้ไขข้อมูลในฟอร์ม
3. คลิก "บันทึก"

**ข.4.3 เปิด/ปิดการเผยแพร่**

คลิกปุ่ม Toggle ในคอลัมน์สถานะ ระบบเปลี่ยนทันที สถานะ "เผยแพร่" (สีเขียว) หมายความว่าผู้เยี่ยมชมมองเห็นข้อมูลของปีนี้ สถานะ "ซ่อน" (สีเทา) หมายความว่าซ่อนข้อมูลทั้งหมดของปีนี้จากสาธารณะ การซ่อนปีจะซ่อนทุกอย่างในปีนั้นพร้อมกัน ทั้ง Banner, Content, รายงาน, ประกาศ และคำสั่ง

**ข.4.4 ลบปี**

1. คลิกไอคอนถังขยะ (ลบ) ในแถวของปีที่ต้องการ
2. ยืนยันในป๊อปอัพ

คำเตือน: การลบปีจะลบข้อมูลทั้งหมดของปีนั้นถาวร ได้แก่ Banner, Content, รายงาน, ประกาศ และคำสั่ง รวมถึงไฟล์ใน Storage ไม่สามารถกู้คืนได้

---

### ข.5 จัดการ Banner

URL: http://localhost/admin/banners/{ปี}

Banner คือรูปภาพที่แสดงใน Slider บนหน้าเว็บ แต่ละ Banner สังกัดปี SCD หนึ่งปี

**ข.5.1 เพิ่ม Banner**

1. เลือกปีจากแถบเลือกปีด้านบน
2. คลิก "เพิ่ม Banner"
3. กรอกข้อมูลดังนี้

| ช่อง | คำอธิบาย | บังคับ |
|------|---------|--------|
| รูปภาพ | อัปโหลด JPG/PNG สูงสุด 100 MB | จำเป็น |
| หมวดหมู่ | 0 = Slider หลัก, 1 = Slider รอง | จำเป็น |
| ประเภทลิงก์ | ไม่มีลิงก์ / URL / PDF | จำเป็น |
| URL | ระบุ URL ปลายทาง (กรณีเลือก URL) | เฉพาะกรณี |
| ไฟล์ PDF | อัปโหลด PDF สูงสุด 100 MB (กรณีเลือก PDF) | เฉพาะกรณี |

4. ใช้ Image Cropper เพื่อปรับขนาดรูปภาพให้พอดีกับ Slider
5. คลิก "บันทึก"

ประเภทลิงก์มี 3 แบบ ได้แก่ ไม่มีลิงก์ (ภาพแสดงเฉยๆ) URL (เมื่อคลิกภาพจะเปิด URL ที่กำหนด) และ PDF (เมื่อคลิกภาพจะเปิดไฟล์ PDF ที่อัปโหลด)

**ข.5.2 แก้ไขและลบ Banner**

การแก้ไข คลิก "แก้ไข" ข้างรายการ แก้ไขข้อมูล แล้วคลิก "บันทึก" หากไม่เลือกรูปใหม่ รูปเดิมจะยังคงอยู่

การลบ คลิก "ลบ" ข้างรายการ แล้วยืนยันในป๊อปอัพ ระบบลบทั้งข้อมูลในฐานข้อมูลและไฟล์รูปภาพ/PDF ใน Storage พร้อมกัน

**ข.5.3 จัดลำดับ Banner**

1. คลิก "จัดลำดับ"
2. ลากรายการ (Drag and Drop) เพื่อเรียงลำดับ
3. คลิก "บันทึกลำดับ"

ลำดับจะส่งผลต่อลำดับภาพใน Slider บนหน้าเว็บทันที

---

### ข.6 จัดการตัวชี้วัด (Content)

URL: http://localhost/admin/contents/{ปี}

Content คือตัวชี้วัดและเนื้อหาหลักของ SCD ที่แสดงในหน้าแรก จัดเก็บเป็นโครงสร้างโฟลเดอร์ (Tree) ดังนี้

```
ตัวชี้วัด A (ระดับหลัก)
    ตัวชี้วัดย่อย A.1
        เอกสาร A.1.1.pdf
        เอกสาร A.1.2.pdf
    เอกสารรวม A.pdf
ตัวชี้วัด B (ระดับหลัก)
    เอกสาร B.pdf
```

การนำทางภายใน Content ทำได้โดยคลิกโฟลเดอร์เพื่อเข้าดูรายการภายใน ดู Breadcrumbs ด้านบนเพื่อรู้ตำแหน่งปัจจุบัน และคลิก Breadcrumb เพื่อย้อนกลับ

**ข.6.1 เพิ่ม Content**

1. เลือกปีจากแถบเลือกปี
2. หากต้องการเพิ่มภายในโฟลเดอร์ ให้เปิดโฟลเดอร์นั้นก่อน
3. คลิก "เพิ่มโฟลเดอร์" หรือ "เพิ่มไฟล์"

สำหรับโฟลเดอร์ ต้องระบุชื่อโฟลเดอร์ สามารถเพิ่มรูปภาพประจำโฟลเดอร์แบบไม่บังคับ และเลือก Toggle แสดง/ซ่อน

สำหรับไฟล์ ต้องระบุชื่อรายการและอัปโหลดไฟล์ PDF (สูงสุด 100 MB) สามารถเพิ่มรูปภาพประกอบแบบไม่บังคับ และเลือก Toggle แสดง/ซ่อน

4. คลิก "บันทึก"

**ข.6.2 แก้ไข ซ่อน/แสดง จัดลำดับ และลบ Content**

การแก้ไข คลิก "แก้ไข" ข้างรายการ แก้ไขข้อมูล แล้วคลิก "บันทึก"

การซ่อน/แสดง คลิกปุ่ม Toggle ในคอลัมน์สถานะ รายการที่ซ่อนยังคงอยู่ในฐานข้อมูล Admin ยังเห็น แต่ผู้เยี่ยมชมจะไม่เห็นบนหน้าเว็บ

การจัดลำดับ คลิก "จัดลำดับ" ลากรายการเพื่อเรียงใหม่ แล้วคลิก "บันทึกลำดับ"

การลบ คลิก "ลบ" ข้างรายการ แล้วยืนยัน คำเตือน: การลบโฟลเดอร์จะลบทุกรายการและไฟล์ภายในโฟลเดอร์นั้นถาวร

### ข.7 จัดการรายงาน SCD

URL: http://localhost/admin/reports/{ปี}

รายงาน SCD คือไฟล์ PDF รายงานผลการจัดอันดับประจำปี แต่ละปีมีได้เพียง 1 ไฟล์

**ข.7.1 อัปโหลดรายงาน**

1. เลือกปีจากแถบเลือกปี
2. คลิก "อัปโหลดรายงาน"
3. เลือกไฟล์ PDF (สูงสุด 100 MB)
4. คลิก "อัปโหลด"

หากปีนั้นมีรายงานแล้ว ต้องลบรายงานเก่าก่อนจึงจะอัปโหลดใหม่ได้

**ข.7.2 ดูสถิติรายงาน**

ตารางรายการแสดงชื่อไฟล์รายงาน จำนวนการดู (View Count) ที่นับโดยอัตโนมัติเมื่อผู้เยี่ยมชมเปิดดู และวันที่อัปโหลด

**ข.7.3 แก้ไขและลบรายงาน**

การแก้ไข คลิก "แก้ไข" เลือกไฟล์ PDF ใหม่ แล้วคลิก "บันทึก" ระบบจะลบไฟล์เก่าออกจาก Storage โดยอัตโนมัติ

การลบ คลิก "ลบ" ข้างรายการ แล้วยืนยัน ระบบลบทั้งข้อมูลและไฟล์ PDF ออกจาก Storage

URL: http://localhost/admin/announcements/{ปี}

ประกาศ (Announcements) จัดเก็บในรูปแบบโครงสร้างโฟลเดอร์ (Tree) สังกัดปี SCD การเพิ่ม แก้ไข ซ่อน/แสดง จัดลำดับ และลบ ทำในลักษณะเดียวกับการจัดการ Content ที่อธิบายในข้อ ข.6

คำสั่ง (Directives) มีการจัดการเหมือนกับประกาศทุกขั้นตอน ต่างกันเพียงเก็บข้อมูลแยกตาราง (scd_directives) เพื่อแยกประเภทเอกสาร และ URL ใช้ /admin/directives/ แทน /admin/announcements/

### ข.10 จัดการโปรไฟล์

URL: http://localhost/admin/profile

**ข.10.1 แก้ไขข้อมูลส่วนตัว**

1. แก้ไขชื่อและ/หรืออีเมล
2. คลิก "บันทึก"

หากเปลี่ยนอีเมล ระบบส่งลิงก์ยืนยันไปยังอีเมลใหม่ ต้องยืนยันก่อนใช้งาน

**ข.10.2 เปลี่ยนรหัสผ่าน**

1. กรอกรหัสผ่านปัจจุบัน
2. กรอกรหัสผ่านใหม่ (ขั้นต่ำ 8 ตัวอักษร)
3. กรอกยืนยันรหัสผ่านใหม่
4. คลิก "เปลี่ยนรหัสผ่าน"

**ข.10.3 ลบบัญชีผู้ใช้**

คำเตือน: การลบบัญชีเป็นการถาวร ไม่สามารถกู้คืนได้ ต้องสร้างบัญชีใหม่ผ่าน IT เท่านั้น ขั้นตอนคือ คลิก "ลบบัญชี" กรอกรหัสผ่านเพื่อยืนยัน แล้วคลิก "ยืนยันการลบ" ระบบออกจากระบบและลบบัญชีอัตโนมัติ

### ข.11 ข้อจำกัดและข้อควรระวัง

**ข.11.1 ขีดจำกัดไฟล์อัปโหลด**

| ประเภทไฟล์ | ขนาดสูงสุด | รูปแบบที่รองรับ |
|------------|----------|----------------|
| รูปภาพ | 100 MB | JPG, PNG, WEBP, GIF |
| ไฟล์ PDF | 100 MB | PDF เท่านั้น |

**ข.11.2 ผลของการลบข้อมูล**

| สิ่งที่ลบ | ผลที่เกิดขึ้น |
|----------|-------------|
| ปี SCD | ลบ Banner, Content, รายงาน, ประกาศ, คำสั่ง ทั้งหมดของปีนั้น รวมถึงไฟล์ใน Storage |
| โฟลเดอร์ | ลบรายการและไฟล์ทั้งหมดภายในโฟลเดอร์ |
| ไฟล์/Banner | ลบข้อมูลและไฟล์จาก Storage ถาวร |
| บัญชีผู้ใช้ | ลบบัญชีถาวร ต้องให้ IT สร้างใหม่ |

ระบบไม่มี Recycle Bin การลบทุกอย่างเป็นถาวร ควร Backup ข้อมูลสำคัญก่อนลบเสมอ

**ข.11.3 ข้อจำกัดด้านบัญชีผู้ใช้**

- ไม่มีการสมัครสมาชิกเอง ต้องให้ IT สร้างผ่าน Command Line
- ไม่มีระบบ Reset Password ผ่านอีเมล ต้องติดต่อ IT เท่านั้น
- บัญชีทุกบัญชีมีสิทธิ์เท่ากัน (Full Admin) ไม่มีการแยก Role

### ค.3 การติดตั้งและรันระบบ


### ค.6 Middleware และความปลอดภัย

**ค.6.1 AdminAuth Middleware**

เมื่อผู้ใช้เข้าถึง Admin route โดยไม่ได้เข้าสู่ระบบ ระบบจะ abort(404) แทนที่จะ redirect ไป login เพื่อซ่อนการมีอยู่ของ Admin Panel จากผู้ไม่ประสงค์ดี

**ค.6.2 AccessLog Middleware**

บันทึก access log ทุก request ไปที่ storage/logs/laravel.log รวมข้อมูล IP, Method, URL, Status code, Response time, Browser และ Device type

**ค.6.3 Rate Limiting**

ระบบ Rate Limiting ของ Laravel ป้องกัน Brute Force การเข้าสู่ระบบ (5 ครั้งต่อนาที)

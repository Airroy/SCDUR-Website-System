## 1. ภาพรวมระบบ

ระบบ SCD เป็นเว็บไซต์สำหรับเผยแพร่และจัดการข้อมูลการจัดอันดับมหาวิทยาลัย ด้านการพัฒนาชุมชนอย่างยั่งยืน (Sustainable Community Development) ของมหาวิทยาลัยราชภัฏพระนครศรีอยุธยา

### โครงสร้างหลักของระบบ

| ส่วน | ผู้ใช้งาน | คำอธิบาย |
|------|-----------|-----------|
| **หน้าบ้าน (Frontend)** | ผู้เยี่ยมชมทั่วไป | ดูข้อมูล ดาวน์โหลดรายงาน อ่านประกาศ |
| **หลังบ้าน (Admin Panel)** | ผู้ดูแลระบบ | เพิ่ม แก้ไข ลบ ข้อมูลทั้งหมดในระบบ |

### URL หลักของระบบ

| URL | คำอธิบาย |
|-----|-----------|
| `http://localhost` | หน้าเว็บหลักสำหรับผู้เยี่ยมชม |
| `http://localhost/login` | หน้าเข้าสู่ระบบ Admin |
| `http://localhost/admin/dashboard` | หน้า Dashboard Admin (ต้อง login ก่อน) |
| `http://localhost:8080` | phpMyAdmin สำหรับจัดการฐานข้อมูล |

## 2. การเข้าถึงระบบ

### ผู้เยี่ยมชมทั่วไป

ไม่ต้องสมัครสมาชิกหรือเข้าสู่ระบบ เพียงเปิดเบราว์เซอร์และไปที่ URL ของเว็บไซต์ได้เลย

**เบราว์เซอร์ที่รองรับ:**
- Google Chrome (แนะนำ)
- Mozilla Firefox
- Microsoft Edge
- Safari

### ผู้ดูแลระบบ (Admin)

ต้องมีบัญชีผู้ใช้ที่สร้างโดยผู้ดูแลระบบ IT เท่านั้น (ไม่มีการลงทะเบียนด้วยตนเอง) จากนั้นเข้าสู่ระบบผ่าน `/login`

> ⚠️ **หมายเหตุด้านความปลอดภัย:** ลิงก์เข้าสู่ระบบ Admin จะไม่แสดงบนหน้าเว็บสาธารณะ ต้องพิมพ์ URL โดยตรง

## 3. คู่มือผู้เยี่ยมชม (Frontend)

### 3.1 หน้าแรก

**URL:** `http://localhost/`

หน้าแรกประกอบด้วยส่วนหลัก ดังนี้:

#### 🖼️ Banner Slider
- แสดงภาพสไลด์อัตโนมัติ (Auto-slide)
- สามารถกดลูกศร **ซ้าย/ขวา** เพื่อเลื่อนภาพเองได้
- กดจุดด้านล่างภาพเพื่อข้ามไปยังสไลด์ที่ต้องการ
- แต่ละภาพอาจมีลิงก์ไปยังเนื้อหาหรือไฟล์ PDF (คลิกที่ภาพเพื่อเปิด)

#### 📢 ประกาศล่าสุด
- แสดงประกาศล่าสุดสูงสุด 6 รายการ
- คลิกที่รายการเพื่ออ่านรายละเอียดหรือดาวน์โหลดไฟล์แนบ

#### 📁 Content Sections
- แสดงหมวดหมู่ข้อมูลย่อยของ SCD
- คลิกที่โฟลเดอร์/หมวดหมู่เพื่อเข้าดูข้อมูลภายใน

#### 🗓️ การเลือกปี SCD
- ด้านบนของหน้า จะมีปุ่มเลือกปี SCD ที่เผยแพร่
- คลิกปีที่ต้องการเพื่อดูข้อมูลของปีนั้น
- URL จะเปลี่ยนเป็น `/scd/{ปี}` เช่น `/scd/2024`

---

### 3.2 หน้า SCD Ranking

**URL:** `http://localhost/scd/{ปี}`  
ตัวอย่าง: `http://localhost/scd/2024`

หน้านี้แสดงข้อมูลการจัดอันดับ SCD ของปีที่เลือก ประกอบด้วย:

- **Banner Slider** เฉพาะปีนั้น
- **Content Sections** แยกตามหมวดหมู่ของปีนั้น
- **ประกาศ** ที่เกี่ยวข้องกับปีนั้น

**วิธีเปลี่ยนปี:**
1. คลิกเมนูเลือกปีด้านบนของหน้า
2. หรือพิมพ์ URL ตรง เช่น `/scd/2023`

---

### 3.3 หน้าประกาศและคำสั่ง

**URL:** `http://localhost/year/{ปี}/announcements-orders`

หน้านี้แสดงประกาศและคำสั่งทั้งหมดในรูปแบบโครงสร้างโฟลเดอร์ (Tree Structure):

#### การนำทาง
- **โฟลเดอร์** (📁) — คลิกเพื่อเปิดและดูรายการภายใน
- **ไฟล์** (📄) — มีปุ่มสำหรับดูและดาวน์โหลด

#### ปุ่มสำหรับไฟล์ PDF
| ปุ่ม | การทำงาน |
|------|-----------|
| 👁️ ดูไฟล์ | เปิดไฟล์ PDF ในหน้าต่างใหม่ |
| ⬇️ ดาวน์โหลด | บันทึกไฟล์ PDF ลงเครื่อง |

> **หมายเหตุ:** บางรายการอาจมีรูปภาพประกอบ แสดงข้างชื่อรายการ

---

### 3.4 หน้าดาวน์โหลดรายงาน

แสดงไฟล์รายงาน SCD (PDF) ที่เผยแพร่ในแต่ละปี

**วิธีดาวน์โหลดรายงาน:**
1. เลือกปีที่ต้องการจากเมนู
2. คลิก **"ดูรายงาน"** เพื่อเปิดดูใน PDF Viewer
3. หรือคลิก **"ดาวน์โหลด"** เพื่อบันทึกไฟล์

---

### 3.5 หน้าเกี่ยวกับหน่วยงาน

แสดงข้อมูลขององค์กร ประวัติความเป็นมา และบทบาทหน้าที่ของหน่วยงาน

---

### 3.6 หน้าติดต่อเรา

แสดงข้อมูลการติดต่อ ประกอบด้วย:
- ที่อยู่หน่วยงาน
- หมายเลขโทรศัพท์
- อีเมล
- แผนที่แสดงที่ตั้ง (Google Maps)

## 4. คู่มือผู้ดูแลระบบ (Admin Panel)

### 4.1 การเข้าสู่ระบบ Admin

**URL:** `http://localhost/login`

#### ขั้นตอนการเข้าสู่ระบบ

1. เปิดเบราว์เซอร์ ไปที่ `http://localhost/login`
2. กรอก **อีเมล** และ **รหัสผ่าน** ในช่องที่กำหนด
3. (ไม่บังคับ) ติ๊ก **"จำฉันไว้"** เพื่อไม่ต้อง login ซ้ำในครั้งต่อไป
4. คลิกปุ่ม **"เข้าสู่ระบบ"**
5. ระบบจะพาไปยังหน้า **Dashboard** โดยอัตโนมัติ

#### การออกจากระบบ

1. คลิกชื่อผู้ใช้หรือเมนูด้านบนขวา
2. คลิก **"ออกจากระบบ"**
3. ระบบจะ redirect กลับไปหน้าเว็บหลัก

> ⚠️ **ระบบป้องกันการเข้าสู่ระบบซ้ำ:** หาก login ผิดพลาดเกิน 5 ครั้ง ระบบจะล็อคการเข้าสู่ระบบชั่วคราว

---

### 4.2 Dashboard

**URL:** `http://localhost/admin/dashboard`

Dashboard แสดงภาพรวมของข้อมูลทั้งหมดในระบบ:

| สถิติ | คำอธิบาย |
|-------|-----------|
| **ปี SCD ทั้งหมด** | จำนวนปี SCD ที่มีในระบบ |
| **ปีที่เผยแพร่** | จำนวนปีที่เปิดให้สาธารณะเข้าชมได้ |
| **รายงานทั้งหมด** | จำนวนไฟล์รายงาน PDF ที่อัปโหลด |
| **Banner ทั้งหมด** | จำนวน Banner รูปภาพในระบบ |

**เมนูนำทาง Admin (Sidebar):**
- 🏠 Dashboard
- 📅 จัดการปี SCD
- 🖼️ จัดการ Banner
- 📁 จัดการ Content
- 📊 จัดการรายงาน
- 📢 จัดการประกาศ
- 📋 จัดการคำสั่ง
- 👤 โปรไฟล์

---

### 4.3 จัดการปี SCD

**URL:** `http://localhost/admin/years`

หน้านี้ใช้สำหรับสร้างและจัดการปี SCD ซึ่งเป็นหน่วยหลักที่ข้อมูลทั้งหมดในระบบอิงอยู่

#### ตารางรายการปี SCD

แสดงรายการปีทั้งหมดพร้อม:
- ชื่อปี (เช่น SCD 2024)
- วันที่สร้าง
- สถานะการเผยแพร่ (เผยแพร่ / ซ่อน)
- ปุ่มแก้ไขและลบ

#### ➕ เพิ่มปีใหม่

1. คลิกปุ่ม **"เพิ่มปีใหม่"** (มุมขวาบน)
2. กรอกข้อมูลในฟอร์ม:
   - **ปี** (ค.ศ.) — เช่น `2024`
   - **วันที่สร้าง** — เลือกวันที่จาก Date Picker
   - **สถานะ** — เลือก "เผยแพร่" หรือ "ซ่อน"
3. คลิก **"บันทึก"**

> **หมายเหตุ:** แต่ละปีต้องไม่ซ้ำกัน — ไม่สามารถเพิ่มปีเดิมได้สองครั้ง

#### ✏️ แก้ไขปี

1. คลิกปุ่ม **"แก้ไข"** (ไอคอนดินสอ) ข้างรายการที่ต้องการ
2. แก้ไขข้อมูลในฟอร์ม
3. คลิก **"บันทึก"**

#### 🔄 เปิด/ปิดการเผยแพร่

1. คลิกปุ่มสลับ (Toggle) ในคอลัมน์สถานะ
2. ระบบจะเปลี่ยนสถานะทันที:
   - **เผยแพร่** (สีเขียว) — ผู้เยี่ยมชมสามารถมองเห็นข้อมูลของปีนี้
   - **ซ่อน** (สีเทา) — ซ่อนข้อมูลของปีนี้จากสาธารณะ

> ⚠️ **ข้อควรระวัง:** การซ่อนปีจะซ่อนเนื้อหาทั้งหมดของปีนั้น ทั้ง Banner, Content, รายงาน, ประกาศ และคำสั่ง

#### 🗑️ ลบปี

1. คลิกปุ่ม **"ลบ"** (ไอคอนถังขยะ) ข้างรายการ
2. อ่านข้อความยืนยันให้ครบ
3. คลิก **"ยืนยันการลบ"**

> ⚠️ **คำเตือน:** การลบปีจะลบข้อมูลทั้งหมดที่เกี่ยวข้อง (Banner, Content, รายงาน, ประกาศ, คำสั่ง) อย่างถาวร ไม่สามารถกู้คืนได้

### 4.4 จัดการ Banner

**URL:** `http://localhost/admin/banners` หรือ `http://localhost/admin/banners/{ปี}`

Banner คือรูปภาพที่แสดงใน Slider บนหน้าแรกของเว็บไซต์

#### ➕ เพิ่ม Banner ใหม่

1. เลือกปีที่ต้องการจากเมนูด้านบน
2. คลิกปุ่ม **"เพิ่ม Banner"**
3. กรอกข้อมูล:

| ช่อง | คำอธิบาย | บังคับ |
|------|-----------|--------|
| **รูปภาพ** | อัปโหลดรูปภาพ (JPG/PNG, สูงสุด 10MB) | ✅ |
| **ประเภทลิงก์** | เลือก: ไม่มีลิงก์ / URL / PDF | ✅ |
| **URL** | ระบุ URL ปลายทาง (ถ้าเลือก URL) | — |
| **ไฟล์ PDF** | อัปโหลด PDF ที่จะเปิดเมื่อคลิกภาพ (ถ้าเลือก PDF) | — |
| **หมวดหมู่** | Slider (0) หรือ อื่นๆ (1) | ✅ |
| **ลำดับ** | ตำแหน่งในการแสดง | ✅ |

4. ใช้ **Image Cropper** เพื่อตัดและปรับขนาดรูปภาพก่อนบันทึก
5. คลิก **"บันทึก"**

#### ✏️ แก้ไข Banner

1. คลิกปุ่ม **"แก้ไข"** ข้างรายการ
2. แก้ไขข้อมูลที่ต้องการ (ถ้าไม่เลือกรูปใหม่ รูปเดิมจะยังคงอยู่)
3. คลิก **"บันทึก"**

#### 🔢 จัดลำดับ Banner

1. คลิกปุ่ม **"จัดลำดับ"**
2. ลาก (Drag) รายการเพื่อเปลี่ยนลำดับการแสดงผล
3. คลิก **"บันทึกลำดับ"**

#### 🗑️ ลบ Banner

1. คลิกปุ่ม **"ลบ"** ข้างรายการ
2. ยืนยันการลบในป๊อปอัพ
3. ระบบจะลบรูปภาพและไฟล์ PDF ที่เกี่ยวข้องออกจากระบบด้วย

### 4.5 จัดการ Content

**URL:** `http://localhost/admin/contents` หรือ `http://localhost/admin/contents/{ปี}`

Content คือข้อมูลหมวดหมู่ย่อยของ SCD ที่แสดงในหน้าแรกและหน้า SCD ของแต่ละปี โครงสร้างเป็นแบบ **Tree (โฟลเดอร์/ไฟล์)**

#### โครงสร้าง Tree

```
📁 โฟลเดอร์หลัก (root)
├── 📁 โฟลเดอร์ย่อย
│   ├── 📄 ไฟล์ PDF
│   └── 📄 ไฟล์ PDF
└── 📄 ไฟล์ PDF
```

#### การนำทางใน Content

- คลิก **โฟลเดอร์** เพื่อเข้าดูข้อมูลภายใน
- ดู **Breadcrumbs** ด้านบนเพื่อรู้ตำแหน่งปัจจุบัน
- คลิก Breadcrumb เพื่อกลับไปยังโฟลเดอร์ก่อนหน้า

#### ➕ เพิ่ม Content

1. เลือกปีและโฟลเดอร์ (ถ้าต้องการ)
2. คลิก **"เพิ่มรายการ"**
3. เลือกประเภท:
   - **📁 โฟลเดอร์** — สร้างหมวดหมู่ใหม่
   - **📄 ไฟล์** — อัปโหลดข้อมูล

**สำหรับโฟลเดอร์:**

| ช่อง | คำอธิบาย |
|------|-----------|
| ชื่อโฟลเดอร์ | ชื่อที่จะแสดง |
| รูปภาพ (ไม่บังคับ) | รูปประจำโฟลเดอร์ (JPG/PNG) |
| แสดง/ซ่อน | Toggle เปิด-ปิดการแสดงผล |

**สำหรับไฟล์:**

| ช่อง | คำอธิบาย |
|------|-----------|
| ชื่อรายการ | ชื่อที่จะแสดง |
| ไฟล์ PDF | อัปโหลดเอกสาร PDF (สูงสุด 100MB) |
| รูปภาพ (ไม่บังคับ) | รูปประจำรายการ (JPG/PNG) |
| แสดง/ซ่อน | Toggle เปิด-ปิดการแสดงผล |

4. คลิก **"บันทึก"**

#### ✏️ แก้ไข Content

1. คลิกปุ่ม **"แก้ไข"** ข้างรายการ
2. แก้ไขข้อมูลที่ต้องการ
3. คลิก **"บันทึก"**

#### 👁️ เปิด/ปิดการแสดงผล

- คลิกปุ่ม Toggle ในคอลัมน์สถานะ เพื่อซ่อนหรือแสดงรายการบนหน้าเว็บสาธารณะ

#### 🔢 จัดลำดับ Content

1. คลิกปุ่ม **"จัดลำดับ"**
2. ลากรายการเพื่อเปลี่ยนลำดับ
3. คลิก **"บันทึกลำดับ"**

#### 🗑️ ลบ Content

1. คลิกปุ่ม **"ลบ"** ข้างรายการ
2. ยืนยันในป๊อปอัพ

> ⚠️ **ข้อควรระวัง:** การลบโฟลเดอร์จะลบรายการทั้งหมดภายในโฟลเดอร์นั้นด้วย (รวมถึงไฟล์ PDF ที่อัปโหลด)

### 4.6 จัดการรายงาน SCD

**URL:** `http://localhost/admin/reports` หรือ `http://localhost/admin/reports/{ปี}`

รายงาน SCD คือไฟล์ PDF รายงานผลการจัดอันดับประจำปี (**แต่ละปีมีได้เพียง 1 ไฟล์**)

#### ➕ อัปโหลดรายงาน

1. เลือกปีที่ต้องการจากเมนู
2. คลิก **"อัปโหลดรายงาน"**
3. เลือกไฟล์ PDF จากเครื่อง (สูงสุด 100MB)
4. คลิก **"อัปโหลด"**

> ถ้าปีนั้นมีรายงานอยู่แล้ว ต้องลบรายงานเก่าก่อนจึงจะอัปโหลดใหม่ได้

#### 📊 ดูสถิติรายงาน

ในตารางรายการ จะแสดง:
- **ชื่อไฟล์** ของรายงาน
- **จำนวนการดู** (View Count)
- วันที่อัปโหลด

#### 🗑️ ลบรายงาน

1. คลิกปุ่ม **"ลบ"** ข้างรายการ
2. ยืนยันในป๊อปอัพ
3. ระบบจะลบไฟล์ PDF ออกจาก Storage ด้วย

---

### 4.7 จัดการประกาศ

**URL:** `http://localhost/admin/announcements` หรือ `http://localhost/admin/announcements/{ปี}`

ประกาศ (Announcements) คือเอกสารประกาศต่างๆ ที่จัดเก็บในรูปแบบ **โครงสร้างโฟลเดอร์ (Tree)**

#### โครงสร้าง Tree (เช่นเดียวกับ Content)

```
📁 ประกาศ ปี 2024
├── 📁 ไตรมาสที่ 1
│   ├── 📄 ประกาศฉบับที่ 1.pdf
│   └── 📄 ประกาศฉบับที่ 2.pdf
└── 📄 ประกาศกลางปี.pdf
```

#### ➕ เพิ่มประกาศ

1. เลือกปีและโฟลเดอร์ (ถ้าต้องการ)
2. คลิก **"เพิ่มประกาศ"**
3. กรอกข้อมูล (เหมือนกับ Content — ดูหัวข้อ 4.5)
4. คลิก **"บันทึก"**

#### ✏️ แก้ไขประกาศ

1. คลิกปุ่ม **"แก้ไข"** ข้างรายการ
2. แก้ไขข้อมูลที่ต้องการ
3. คลิก **"บันทึก"**

#### 👁️ เปิด/ปิดการแสดงผล

- คลิก Toggle ในคอลัมน์สถานะเพื่อซ่อนหรือแสดงประกาศ

#### 🔢 จัดลำดับประกาศ

1. คลิกปุ่ม **"จัดลำดับ"**
2. ลากเปลี่ยนลำดับ
3. คลิก **"บันทึกลำดับ"**

#### 🗑️ ลบประกาศ

1. คลิก **"ลบ"** ข้างรายการ
2. ยืนยันในป๊อปอัพ

### 4.8 จัดการคำสั่ง

**URL:** `http://localhost/admin/directives` หรือ `http://localhost/admin/directives/{ปี}`

คำสั่ง (Directives) มีโครงสร้างและการจัดการเหมือนกับ **ประกาศ** ทุกอย่าง เพียงแต่เก็บข้อมูลแยกตาราง (scd_directives) เพื่อแยกประเภทเอกสาร

> การใช้งานทำเหมือนกับการจัดการประกาศในหัวข้อ 4.7 ทุกขั้นตอน

### 4.9 จัดการโปรไฟล์

**URL:** `http://localhost/admin/profile` (เข้าถึงได้จากเมนูด้านบนขวา)

#### 👤 แก้ไขข้อมูลส่วนตัว

1. กรอก **ชื่อ** ที่ต้องการแก้ไข
2. กรอก **อีเมล** ที่ต้องการแก้ไข
3. คลิก **"บันทึก"**

> หากเปลี่ยนอีเมล ระบบจะส่งลิงก์ยืนยันอีเมลใหม่ให้

#### 🔒 เปลี่ยนรหัสผ่าน

1. กรอก **รหัสผ่านปัจจุบัน**
2. กรอก **รหัสผ่านใหม่** (ขั้นต่ำ 8 ตัวอักษร)
3. กรอก **ยืนยันรหัสผ่านใหม่** อีกครั้ง
4. คลิก **"เปลี่ยนรหัสผ่าน"**

**ข้อกำหนดรหัสผ่าน:**
- ขั้นต่ำ 8 ตัวอักษร
- รหัสผ่านใหม่ต้องตรงกันทั้งสองช่อง
- รหัสผ่านปัจจุบันต้องถูกต้อง

#### ⚠️ ลบบัญชีผู้ใช้

> ⚠️ **คำเตือน:** การลบบัญชีเป็นการดำเนินการถาวร ไม่สามารถกู้คืนได้

1. คลิกปุ่ม **"ลบบัญชี"** ด้านล่างของหน้า
2. กรอกรหัสผ่านเพื่อยืนยัน
3. คลิก **"ยืนยันการลบ"**
4. ระบบจะออกจากระบบและลบบัญชีอัตโนมัติ

## 5. ข้อจำกัดและข้อควรระวัง

### ข้อจำกัดไฟล์อัปโหลด

| ประเภทไฟล์ | ขนาดสูงสุด | รูปแบบที่รองรับ |
|------------|------------|-----------------|
| รูปภาพ | 10 MB | JPG, PNG |
| ไฟล์ PDF | 100 MB | PDF เท่านั้น |

### ข้อควรระวังในการลบข้อมูล

| อ่านก่อนลบ | ผลที่เกิดขึ้น |
|-----------|----------------|
| ลบ **ปี SCD** | ลบ Banner, Content, รายงาน, ประกาศ, คำสั่ง ทั้งหมดของปีนั้น |
| ลบ **โฟลเดอร์** | ลบรายการและไฟล์ทั้งหมดภายในโฟลเดอร์ |
| ลบ **ไฟล์/Banner** | ลบไฟล์จาก Storage ถาวร |
| ลบ **บัญชีผู้ใช้** | ลบบัญชีถาวร ต้องสร้างใหม่ผ่าน Command Line |

### ข้อจำกัดด้านบัญชีผู้ใช้

- **ไม่มีการสมัครสมาชิกเอง** — ต้องให้ผู้ดูแลระบบ IT สร้างบัญชีผ่าน Command Line
- **ไม่มีฟีเจอร์ลืมรหัสผ่าน (Reset Password ผ่านอีเมล)** — ต้องติดต่อ IT เพื่อ Reset
- **บัญชี Admin มีสิทธิ์เท่ากันทั้งหมด** — ไม่มีระบบ Role/Permission แยก

## 6. การแก้ไขปัญหาเบื้องต้น

### ปัญหา: รูปภาพไม่แสดงผล

**สาเหตุ:** Storage symlink อาจขาด  
**วิธีแก้:** แจ้ง IT เพื่อรันคำสั่ง:
```bash
./vendor/bin/sail artisan storage:link
```

---

### ปัญหา: อัปโหลดไฟล์ไม่สำเร็จ

**ตรวจสอบ:**
1. ขนาดไฟล์เกิน 100MB หรือเปล่า?
2. นามสกุลไฟล์ถูกต้องหรือเปล่า? (ต้องเป็น PDF, JPG, หรือ PNG เท่านั้น)
3. การเชื่อมต่ออินเทอร์เน็ตปกติหรือเปล่า?

---

### ปัญหา: ข้อมูลไม่อัปเดตหลังบันทึก

**วิธีแก้:**
1. กด `Ctrl + F5` หรือ `Cmd + Shift + R` เพื่อ Reload หน้าใหม่แบบ Hard Refresh
2. หากยังไม่หาย แจ้ง IT เพื่อ Clear Cache:
```bash
./vendor/bin/sail artisan cache:clear
./vendor/bin/sail artisan view:clear
```

---

### ปัญหา: เข้าหน้า Admin ไม่ได้ (แสดง 404)

**สาเหตุที่เป็นไปได้:**
- ยังไม่ได้ login
- Session หมดอายุ

**วิธีแก้:**
1. ไปที่ `http://localhost/login`
2. Login ใหม่อีกครั้ง

---

### ปัญหา: Login ไม่ได้ (Too Many Attempts)

**สาเหตุ:** พยายาม Login ผิดพลาดมากกว่า 5 ครั้ง  
**วิธีแก้:** รอ 1 นาทีแล้วลองใหม่ หรือแจ้ง IT เพื่อ Clear Rate Limit:
```bash
./vendor/bin/sail artisan cache:clear
```

---

### ปัญหา: เว็บไซต์ขึ้น Error 500

**วิธีแก้:** แจ้ง IT เพื่อตรวจสอบ logs:
```bash
./vendor/bin/sail logs
```
หรือดูใน `storage/logs/laravel.log`

## 7. คำถามที่พบบ่อย

**Q: ฉันสามารถเพิ่มบัญชี Admin เองได้ไหม?**  
A: ไม่ได้ ต้องให้ผู้ดูแลระบบ IT รันคำสั่ง `./vendor/bin/sail artisan admin:create` เท่านั้น ระบบไม่มีฟีเจอร์ลงทะเบียนเอง

---

**Q: ถ้าลืมรหัสผ่าน Admin ต้องทำอย่างไร?**  
A: ติดต่อผู้ดูแลระบบ IT เพื่อรีเซ็ตรหัสผ่านผ่าน Command Line หรือสร้างบัญชีใหม่

---

**Q: สามารถเข้า Admin จากนอกองค์กรได้ไหม?**  
A: ขึ้นอยู่กับการตั้งค่า Network ของระบบ ปกติจะต้องเข้าผ่าน VPN หรือภายใน Network ขององค์กรเท่านั้น

---

**Q: ปีที่ "ซ่อน" ยังสามารถแก้ไขได้ใน Admin ไหม?**  
A: ได้ การซ่อนปีเป็นเพียงการซ่อนจากผู้เยี่ยมชม Admin ยังสามารถแก้ไขข้อมูลได้ตามปกติ

---

**Q: ไฟล์ที่ลบออกไปแล้วสามารถกู้คืนได้ไหม?**  
A: ไม่ได้ การลบไฟล์เป็นการดำเนินการถาวร ระบบไม่มี Recycle Bin ควร Backup ข้อมูลสำคัญก่อนลบ

---

**Q: Banner Slider สามารถเพิ่มได้มากแค่ไหน?**  
A: ไม่จำกัดจำนวน แต่แนะนำให้มีไม่เกิน 5-7 ภาพเพื่อ UX ที่ดี

---

**Q: ทำไมรูปภาพ Banner ต้องผ่าน Image Cropper?**  
A: เพื่อให้รูปภาพมีขนาดและสัดส่วนที่เหมาะสมกับ Slider โดยอัตโนมัติ ป้องกันภาพผิดสัดส่วน

---

**Q: การดาวน์โหลดไฟล์มีการนับสถิติหรือเปล่า?**  
A: ใช่ ระบบนับทั้ง **View Count** (การดูออนไลน์) และ **Download Count** (การดาวน์โหลด) สำหรับไฟล์ประกาศ คำสั่ง และรายงาน
